# 网络优化说明

## 问题描述
应用在国内网络环境下首次加载时出现白屏时间过长的问题，主要原因是：
1. GitHub 原始文件访问速度慢
2. 代理服务不稳定
3. 网络超时时间过长
4. 缺少离线支持

## 优化措施

### 1. CDN 优化
- **主要地址**：使用 `cdn.jsdelivr.net` 作为主要的基础日程库获取地址
- **备用地址**：按国内网络友好程度排序：
  - `fastly.jsdelivr.net`
  - `gcore.jsdelivr.net` 
  - `gh-proxy.org`
  - `raw.githubusercontent.com`

### 2. 网络请求优化
- **并发请求**：同时尝试前两个最快的 CDN 地址
- **超时控制**：将超时时间从 8 秒减少到 5 秒
- **缓存策略**：将缓存时间从 2 小时延长到 6 小时，减少网络请求频率

### 3. 离线支持
- **网络状态检测**：只在在线时尝试更新基础日程库
- **离线模式**：网络断开时使用本地缓存数据
- **状态提示**：显示网络连接状态给用户

### 4. DNS 预解析
在 `index.html` 中添加了关键域名的 DNS 预解析：
- `cdn.jsdelivr.net`
- `fastly.jsdelivr.net`
- `gcore.jsdelivr.net`
- `gh-proxy.org`
- `raw.githubusercontent.com`

### 5. 构建优化
- **代码分割**：将第三方库单独打包
- **压缩优化**：移除 console 和 debugger
- **资源预连接**：预连接到关键 CDN 服务

### 6. 用户体验优化
- **立即显示**：优先使用缓存数据立即显示界面
- **后台更新**：在后台异步更新基础日程库
- **网络状态提示**：显示网络连接状态和离线模式提示

## 预期效果
- 首次加载时间显著减少
- 国内网络环境下的访问速度提升
- 网络不稳定时的用户体验改善
- 离线情况下仍可正常使用基本功能

## 技术细节
- 使用 `Promise.allSettled` 并发请求多个 CDN
- 实现智能降级策略，优先使用最快的 CDN
- 添加网络状态监听，自动适应网络环境变化
- 优化缓存策略，减少不必要的网络请求
