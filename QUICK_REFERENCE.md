## 常用操作

### 首次使用

```
1. 打开应用 → 自动加载基础日程库
2. 开始使用 → 标记完成、添加备注
3. 定期备份 → 导出用户数据
```

### 更换设备

```
旧设备：
1. 设置 → 数据管理 → 导出用户数据
2. 保存文件：user-data-xxx.json

新设备：
1. 打开应用 → 自动加载基础日程库
2. 设置 → 数据管理 → 导入用户数据
3. 选择文件 → 完成
```

### 定期维护

```
每周/每月：
1. 导出用户数据备份
2. 检查基础日程库版本
3. 手动更新基础日程库（可选）
```

## 数据流向

### 启动流程

```
打开应用
  ↓
尝试从 GitHub 加载基础日程库
  ↓
成功？
├─ 是 → 缓存到本地
└─ 否 → 使用本地缓存
  ↓
加载用户数据 (localStorage)
  ↓
合并数据
  ↓
显示完整日程
```

### 保存流程

```
用户操作（标记完成、添加备注等）
  ↓
更新 schedules 状态
  ↓
分离数据：
├─ 基础日程 → 只保存修改部分
└─ 用户日程 → 保存完整数据
  ↓
保存到 localStorage (USER_DATA_KEY)
```

### 导出流程

```
点击"导出用户数据"
  ↓
遍历所有日程
  ↓
提取用户数据：
├─ 基础日程 → 只提取修改部分
└─ 用户日程 → 提取完整数据
  ↓
生成 JSON 文件
  ↓
下载：user-data-xxx.json
```

### 导入流程

```
选择 JSON 文件
  ↓
解析文件
  ↓
检测格式：
├─ 新格式（对象）→ 直接使用
└─ 旧格式（数组）→ 自动转换
  ↓
合并用户数据：
├─ 新数据 → 添加
├─ 已存在 → 更新
└─ 备注 → 智能合并
  ↓
保存到 localStorage
  ↓
重新加载并合并显示
```

## 故障排除

| 问题 | 原因 | 解决方案 |
|------|------|---------|
| 基础日程库加载失败 | 网络问题 | 使用缓存数据，稍后手动更新 |
| 数据没有更新 | 浏览器缓存 | 手动更新基础日程库 |
| 用户数据丢失 | localStorage 被清除 | 从备份文件导入 |
| 导入失败 | 文件格式错误 | 检查 JSON 格式是否正确 |
