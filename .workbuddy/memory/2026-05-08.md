## 修复：B站链接跳转"视频已失效"问题

**文件：** `src/components/ScheduleCard.jsx`

**根因：** "跳转链接"和"官方录播"按钮使用 `a.rel = 'noopener noreferrer'`，其中 `noreferrer` 移除了 HTTP Referer 头。B站视频页需要 Referer 头验证来源，缺少时返回"视频已失效"。

**修复：** 参考 `C:\Tools\Elements\Evelynall\枝江日程表\枝江日程表-快捷记录插件\test.html` 测试页中的有效方案。

**根因确认：** B站对无 `share_source` 参数的外部直接跳转有安全限制。

**最终修复方案：** "跳转链接"和"官方录播"按钮统一使用 `window.open(url, '_blank')`，与直播间/动态按钮行为完全一致。不加任何额外参数或 referrer 处理，保持和 test.html 一致。
